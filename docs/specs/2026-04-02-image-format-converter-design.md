# 图片格式互转小程序设计文档

日期：2026-04-02

## 1. 目标

设计并实现一个 Windows 桌面 `exe` 小程序，用于常见图片格式之间的批量互转。程序面向本地使用场景，重点是操作直接、界面简单、支持拖拽导入、默认输出路径可见且可修改。

第一版目标：

- 支持拖拽或按钮导入多张图片
- 支持批量转换
- 支持设置并保存默认输出路径
- 主界面直接显示当前输出路径
- 转换前允许临时修改输出路径
- 支持常见图片格式互转
- 打包为 Windows 双击可运行的 `exe`

## 2. 技术方案

推荐技术栈：

- `Python`
- `PySide6`：构建桌面图形界面
- `Pillow`：执行图片读取、格式转换和保存
- `PyInstaller`：打包为 Windows `exe`

选择理由：

- 开发成本低，适合快速交付桌面工具
- 拖拽、文件选择、路径选择、列表展示都能稳定实现
- 图片格式转换能力足够覆盖本项目第一版需求
- 打包流程成熟，适合交付给非开发用户直接使用

不采用的方案：

- `C# + WinForms/WPF`：Windows 原生程度高，但整体开发成本更重
- `Electron`：界面灵活，但体积大、启动慢，不适合小工具

## 3. 功能范围

### 3.1 输入格式

第一版支持以下输入格式：

- `JPG`
- `JPEG`
- `PNG`
- `BMP`
- `WEBP`
- `GIF`

其中 `GIF` 仅按静态图处理，不保留动画。

### 3.2 输出格式

第一版支持以下输出格式：

- `PNG`
- `JPG`
- `BMP`
- `WEBP`
- `ICO`

### 3.3 核心能力

- 单次导入多张图片
- 批量转换为同一种目标格式
- 自动创建输出目录
- 自动处理输出重名
- 转换完成后提示成功和失败统计
- 支持一键打开输出目录

## 4. 界面设计

整体采用窄版单窗口布局，风格偏轻量工具，不做复杂导航。

### 4.1 主界面区域

主界面包含以下区域：

1. 输出路径区
   - 直接显示当前输出路径
   - 提供“更改路径”按钮
   - 路径信息不隐藏，始终在主界面可见

2. 拖拽导入区
   - 支持将图片直接拖入
   - 支持点击按钮手动添加图片
   - 明确提示支持多张图片批量导入

3. 待转换列表区
   - 显示待转换文件名
   - 可显示图片尺寸和原格式
   - 转换失败项需要可见地标识出来

4. 操作区
   - 提供“输出格式”下拉框
   - 提供“开始转换”按钮
   - 转换完成后显示结果提示，并支持打开输出目录

### 4.2 交互特点

- 主界面宽度偏窄，符合“小工具”视觉定位
- 不使用多页面跳转
- 所有关键操作都在一个窗口内完成
- 输出路径始终可见，避免用户找不到保存位置

## 5. 路径与配置策略

### 5.1 默认输出路径

- 程序首次启动时，引导用户选择一个默认输出路径
- 默认输出路径会持久化保存
- 程序后续启动时自动载入该默认路径并展示在主界面上

### 5.2 临时修改输出路径

- 每次转换前都允许用户临时切换输出路径
- 临时切换可以只作用于当前批次
- 界面实现上应保留是否覆盖默认路径的能力，但第一版也可以先简化为：
  - “更改路径”默认直接更新当前使用路径
  - 并同步写回为新的默认路径

如果实现复杂度允许，可进一步拆为：

- “临时使用此路径”
- “设为默认路径”

第一版优先保证路径清晰可见、可改、可保存。

## 6. 转换规则

### 6.1 批量转换

- 用户一次选择一个目标格式
- 当前批次内所有图片统一转换为该格式

### 6.2 透明通道处理

- 如果源图包含透明通道，而目标格式选择 `JPG`
- 程序在保存前自动铺白底
- 避免出现保存失败或黑底图像

### 6.3 GIF 处理

- `GIF` 只读取静态内容，不保留动画
- 界面和说明文案中需要明确该限制

### 6.4 ICO 处理

- 输出为 `ICO` 时生成通用图标尺寸
- 优先满足“可正常作为图标使用”这一目标

### 6.5 文件命名

- 输出文件默认沿用原文件名
- 只替换扩展名
- 如果目标目录已有重名文件，自动追加序号，例如：
  - `image.png`
  - `image (1).png`
  - `image (2).png`

## 7. 异常处理

### 7.1 导入阶段

- 非支持格式文件拖入时，直接跳过
- 向用户提示“不是支持的图片格式”

### 7.2 转换阶段

- 单张图片转换失败时，不中断整批任务
- 继续处理剩余图片
- 在最终结果中汇总成功数和失败数

### 7.3 失败反馈

失败项需要在列表中标识，并显示简短原因，例如：

- 文件损坏
- 不支持的格式
- 读取失败
- 保存失败

### 7.4 输出目录处理

- 输出目录不存在时自动创建
- 无权限写入时弹出错误提示并停止本次转换

## 8. 典型使用流程

1. 用户打开程序
2. 主界面显示当前默认输出路径
3. 用户拖入多张图片，或点击“添加图片”
4. 用户在下拉框中选择目标格式
5. 如有需要，用户修改输出路径
6. 用户点击“开始转换”
7. 程序执行批量转换
8. 程序显示“成功 X 张，失败 Y 张”
9. 用户可点击打开输出目录查看结果

## 9. 打包与交付

交付目标：

- 输出为 Windows 可直接双击运行的 `exe`
- 用户不需要手动安装 Python
- 程序资源路径和配置文件读写要兼容打包环境

建议使用 `PyInstaller` 打包，并验证：

- 首次启动正常
- 拖拽导入正常
- 配置保存正常
- 转换和打开输出目录正常

## 10. 测试范围

第一版至少覆盖以下验证点：

### 10.1 核心交互

- 拖拽导入图片
- 按钮添加图片
- 批量转换
- 路径显示与修改
- 默认路径保存与重启恢复

### 10.2 文件处理

- 输出目录自动创建
- 重名文件自动追加编号
- 转换后扩展名正确

### 10.3 格式验证

- 透明 `PNG` 转 `JPG`
- `WEBP` 转 `PNG`
- `GIF` 静态转 `PNG`
- `JPG` 转 `ICO`

### 10.4 异常验证

- 导入非图片文件
- 导入损坏图片
- 单张失败不影响整批

## 11. 后续扩展方向

第一版不强制实现，但后续可以考虑：

- 图片预览缩略图
- 每张图片单独删除
- 转换质量或压缩参数设置
- 转换完成自动清空列表
- 深色模式
- 更多格式支持，例如 `TIFF`

## 12. 当前结论

该项目按“轻量桌面小工具”定位实现，核心原则如下：

- 一个窗口完成全部操作
- 默认输出路径直接展示，不隐藏
- 支持拖拽、多图批量、常见格式互转
- 先保证稳定和易用，再考虑高级功能
